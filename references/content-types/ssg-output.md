# Static Site Generator Output

Pre-built output from static site generators. The content is already HTML — the prep work is ensuring paths and structure are compatible with subdirectory hosting.

## What this covers

Build output from: Next.js (static export), Astro, Hugo, Jekyll, Eleventy (11ty), Gatsby, Nuxt (static), VitePress, Vite (plain), Qwik, VuePress, Middleman.

## Identifying the output directory

If the user points to a project root (not a build output), find the output directory:

| Config file present | SSG | Output dir | Build command |
|--------------------|-----|------------|---------------|
| `next.config.*` | Next.js | `out/` | `next build` (with `output: 'export'`) |
| `astro.config.*` | Astro | `dist/` | `astro build` |
| `hugo.toml` / `hugo.yaml` / `config.toml` | Hugo | `public/` | `hugo` |
| `_config.yml` (with Jekyll gems) | Jekyll | `_site/` | `jekyll build` |
| `.eleventy.js` / `eleventy.config.*` | 11ty | `_site/` | `eleventy` |
| `gatsby-config.*` | Gatsby | `public/` | `gatsby build` |
| `nuxt.config.*` | Nuxt 3 | `.output/public/` | `nuxt generate` |
| `.vitepress/config.*` | VitePress | `.vitepress/dist/` | `vitepress build` |
| `docusaurus.config.*` | Docusaurus | `build/` | `docusaurus build` |
| `vite.config.*` (no framework) | Vite | `dist/` | `vite build` |
| `mkdocs.yml` | MkDocs | `site/` | `mkdocs build` |

If the output directory doesn't exist, the project hasn't been built yet. Run the build command first.

## Preparation checklist

### 1. Base path configuration

Every SSG has its own config key for the base path. This must be set **before building** — it affects how the SSG generates asset URLs in the HTML output.

| SSG | Config key | Value for upublish |
|-----|-----------|-------------------|
| Next.js | `basePath` + `assetPrefix` | `basePath: '/{slug}'`, `assetPrefix: '/{slug}/'` |
| Astro | `base` | `base: '/{slug}'` or `base: './'` |
| Hugo | `baseURL` | `baseURL: 'https://{address}.upubli.sh/{slug}/'` |
| Jekyll | `baseurl` | `baseurl: '/{slug}'` |
| 11ty | Manual (no built-in) | Use `pathPrefix` filter or global data |
| Gatsby | `pathPrefix` | `pathPrefix: '/{slug}'` + build with `--prefix-paths` |
| VitePress | `base` | `base: '/{slug}/'` |
| Docusaurus | `url` + `baseUrl` | `url: 'https://{address}.upubli.sh'`, `baseUrl: '/{slug}/'` |
| Vite | `base` | `base: './'` |
| MkDocs | `site_url` | `site_url: 'https://{address}.upubli.sh/{slug}/'` |

**If the user doesn't control the build config** (they're publishing someone else's pre-built output), scan the HTML for absolute paths and fix them post-build.

### 2. Trailing slash behavior

SSGs produce two different file structures, and they interact differently with CDN hosting:

| Structure | Example | URL behavior |
|-----------|---------|-------------|
| Flat files | `about.html` | Accessed at `/about.html` or `/about` (depends on host) |
| Directory indexes | `about/index.html` | Accessed at `/about/` |

| SSG | Default output style |
|-----|---------------------|
| Hugo | Always `about/index.html` |
| Gatsby | Always `about/index.html` |
| Astro | `about/index.html` (configurable: `build.format`) |
| 11ty | `about/index.html` (configurable per template) |
| Next.js | `about.html` (default), `about/index.html` with `trailingSlash: true` |
| Jekyll | `about.html` (flat) |
| Docusaurus | `about/index.html` (configurable: `trailingSlash`) |

**Recommendation:** Directory-style (`page/index.html`) is more portable across hosts. If the SSG supports it, use it.

### 3. Asset subdirectories

SSGs place hashed assets in specific subdirectories within the output:

| SSG | Assets location |
|-----|----------------|
| Astro | `_astro/` |
| Next.js | `_next/static/` |
| Gatsby | `static/` + `page-data/` |
| VitePress | `assets/` |
| Hugo | Mirrors source `static/` dir |
| Vite | `assets/` |

These paths are generated by the build tool and should not be modified. If the base path is set correctly, all references to these directories will be correct.

### 4. Post-build verification

After building with the correct base path, verify the output:

```bash
# Check for absolute paths that should be relative
grep -rn 'src="\/' <output-dir> --include="*.html"
grep -rn 'href="\/' <output-dir> --include="*.html"

# Check for the base path in generated assets
grep -rn '/{slug}/' <output-dir> --include="*.html" | head -5

# Verify index.html exists at root
ls <output-dir>/index.html
```

### 5. Jekyll-specific: underscore directories

Jekyll ignores directories starting with `_` by default. If the SSG output contains directories like `_astro/` or `_next/`, and the content passes through a Jekyll-like processor, add a `.nojekyll` file to the root. This is primarily a GitHub Pages concern but worth noting.

## Common issues

| Issue | Symptom | Fix |
|-------|---------|-----|
| Wrong output directory | Publishing source code instead of build output | Find the correct output dir from the table above |
| Base path not set | Assets 404, broken styles/scripts | Set base path config and rebuild |
| Trailing slash mismatch | Relative paths break on some pages | Ensure SSG and host agree on trailing slash behavior |
| `assetPrefix` missing (Next.js) | Pages load but assets 404 | Set both `basePath` and `assetPrefix` |
| Build not run | No output directory exists | Run the build command first |
| Gatsby `pathPrefix` without `--prefix-paths` | Prefix ignored at build | Add `--prefix-paths` flag to build command |
